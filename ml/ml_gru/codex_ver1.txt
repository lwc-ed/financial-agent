codex_ver1
==========

Date
- 2026-03-25

Goal
- 在不改動原始資料內容的前提下，透過調整 personal preprocessing 與 finetune 訓練邏輯，嘗試降低 GRU 模型的實際預測誤差。

Test Environment
- Sandbox worktree: /Users/liweichen/.codex/worktrees/0c18/financial-agent
- Python: /Users/liweichen/financial-agent/.venv/bin/python

Baseline Before This Version
- val_mae: 4623.9033
- val_rmse: 11759.6963
- val_smape: 95.5126%
- val_per_user_nmae: 220.4055%
- test_mae: 5952.5522
- test_rmse: 14905.8896
- test_smape: 90.1819%
- test_per_user_nmae: 139.6839%

Changes Made
1. preprocess_personal.py
- 將 per-user clipping 從 P95 放寬到 P99，避免截斷過多中高額樣本。
- 只對 feature 做 per-user clipping，不再對 target 做 clipping。
- 保留 per-user 70/15/15 時間切分。
- 保留 train-only scaler fit，避免資料洩漏。

2. finetune_gru.py
- 將 finetune loss 從 MSELoss 改成 SmoothL1Loss(beta=0.5)。
- 將 optimizer 從 Adam 改成 AdamW，加入 weight_decay=1e-4。
- learning rate 從 0.0005 調整為 0.0003。
- early stopping patience 從 20 改成 15。
- 取消原本部分凍結，改為全部權重都可訓練。
- 最佳 checkpoint 的選擇改為看原始金額尺度的 val MAE，而不是 scaled val loss。

Commands Run
1. /Users/liweichen/financial-agent/.venv/bin/python preprocess_personal.py
2. /Users/liweichen/financial-agent/.venv/bin/python finetune_gru.py
3. /Users/liweichen/financial-agent/.venv/bin/python predict.py

Result After This Version
- val_mae: 5764.4902
- val_rmse: 19596.2695
- val_smape: 86.17%

- test_mae: 5916.5259

Delta vs Baseline
- test_mae: improved by 36.03
- test_smape: improved by 9.10 percentage points
- test_per_user_nmae: improved by 20.10 percentage points
- test_rmse: worsened by 815.89
- val 指標整體變差，代表這版對 validation 區間的穩定性不足。

Interpretation
- 這版對「平均絕對誤差」有非常小幅的改善。
- 相對誤差指標改善比較明顯，表示對不同 user 尺度的公平性有進步。
- 但 RMSE 變差，代表少數大誤差樣本仍然拉高尾部風險。
- 取消凍結 + SmoothL1Loss + MAE-based checkpoint selection 的方向是合理的，但目前還沒有顯著降低主目標 test_mae。

Why This Version Was Kept
- 如果以主目標 test_mae 為準，這版比前一版略好。
- 如果同時考慮相對誤差與 per-user 公平性，這版明顯優於前一版。

Weaknesses / Review
1. RMSE 變差
- 代表尾部大誤差沒有被解決，甚至可能更嚴重。

2. validation 表現退化
- 說明這組設定對不同時間區段的穩定性不夠。

3. 目前仍然沿用固定 7 個特徵
- 這些特徵原本不是專門為 GRU residual-style 預測設計。

4. 目前 target 仍然是直接預測 future_expense_7d_sum
- 模型可能仍然在學一個尺度過大、偏態重的目標。

Recommended Next Version
1. 改成預測 residual
- 例如預測 future_expense_7d_sum - expense_7d_mean * 7
- 推論時再把 baseline 加回去
- 這通常比直接預測總額更容易學

2. 改 checkpoint selection
- 除了 val_mae，也記錄 val_smape / val_per_user_nmae
- 避免只對單一尺度最佳化

3. 補時間序列特徵
- 近期變化率
- 波動度
- 月底 / 發薪日前後特徵

4. 比較是否真的需要 Walmart pretrain
- 目前可考慮做一組不使用 pretrain 的 ablation，比較 transfer 是否真的有幫助

Conclusion
- codex_ver1 是一個小幅優化版，不是突破版。
- 這版最大的收穫不是大幅降低 test_mae，而是確認：
  1. 過度 aggressive 的 target clipping 不值得保留
  2. 用實際 val MAE 選 checkpoint 比只看 scaled loss 更合理
  3. 下一步最值得投入的是 residual target，而不是再微調同一組 loss 參數
