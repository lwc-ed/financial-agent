一、本版本改動摘要（相對 V2）
----------------------------------------------------------------
V2 的最大問題：Test Per-user NMAE 異常飆至 410%，
代表某些用戶的相對誤差極高。

V3 針對相對誤差做出改動（沿用 V2 架構 hidden=64）：

  1. Log1p 目標轉換（最核心改動）
     理由：個人消費金額分布高度右偏（right-skewed）。
           原始空間中，模型被大額消費支配，對小額消費
           預測比例失準，導致 SMAPE/NMAE 偏高。
           Log1p 轉換讓分布更對稱：
             原始標準差 16,446 元 → Log1p 標準差 3.26
           訓練完後反轉換（expm1）回實際金額。

     實際分布改變：
       y_train 實際值範圍   : [0.0, 121,867 元]
       y_train log1p 範圍  : [0.000, 11.711]
       標準差：16,446 → 3.26（更均勻，對 SMAPE 有利）

  2. Gaussian 輸入噪聲增強（σ=0.02）
     理由：只有 4,067 筆訓練資料，模型容易記憶特定輸入模式。
           在標準化特徵上加入均值 0、標準差 0.02 的噪聲，
           相當於為每筆資料產生無限多個略有差異的版本，
           提高訓練多樣性，降低過擬合。
           噪聲只在訓練時加入，驗證/測試不加。

  沿用 V2 的機制：
  - hidden=64, Attention, FC(64→32→1)
  - Gradual Unfreezing（Phase 1 凍結 GRU，Phase 2 全量微調）
  - AdamW, weight_decay=5e-4, HuberLoss
  - 使用 V2 pretrain 權重

三、分析與檢討
================================================================

【正面發現】

  1. Per-user NMAE 達到目前全版本最佳
     Test Per-user NMAE：V2 的 410% → V3 的 61.4%（▼85%！）
     Val Per-user NMAE：V2 的 196% → V3 的 73%（▼63%！）
     Log1p 轉換直接解決了 V2 的相對誤差異常問題。
     原因：在 log 空間中，大額消費（如 100,000 元）和小額消費（如 100 元）
           的預測難度更均等，不再讓模型偏重大額用戶的學習。

【問題發現】

  1. Test MAE 繼續退步（6,347 元，比 V0 的 5,952 高 6.6%）
     Log1p 轉換在 log 空間最小化誤差，但轉換回實際金額後，
     因為 expm1 的非線性放大效應，小誤差在高消費區間會變成大誤差。
     這是 log1p 轉換的固有 trade-off：
       改善相對誤差（SMAPE, NMAE）→ 犧牲絕對誤差（MAE, RMSE）

  2. Test SMAPE 退步（91.6%，比 V0 還差）
     這是出乎意料的結果。Log1p 應該有助於 SMAPE，
     但 V3 的 SMAPE 比 V0 的 90.18% 還要差。
     可能原因：Log1p 轉換後，模型在 log 空間的誤差雖然均勻，
               但反轉換後對接近零的預測值特別敏感。
               （SMAPE 計算：|pred-true| / ((|pred|+|true|)/2)
                當 pred≈0 且 true>0 時，SMAPE=200%！）
     更深層原因：模型可能對「零花費日」的預測偏低（趨近 log 空間的 0），
               但真實值有時有大額消費，造成 SMAPE 惡化。

  3. RMSE 明顯退步（17,265 vs V0 的 14,906）
     同上，log 空間的優化讓高消費的大誤差在反轉換後被放大。

================================================================
四、根本原因分析
================================================================

V3 揭示了 Log1p 轉換的核心矛盾：

  Log1p 改善的是「比例誤差」（Per-user NMAE）
  Log1p 犧牲的是「絕對誤差」（MAE, RMSE）
  Log1p 對 SMAPE 的影響不確定（取決於預測值的分布）

  V3 的 Per-user NMAE (61%) 是目前所有版本中最佳，
  但 MAE 和 SMAPE 同時退步，說明不能單純用 log1p 解決問題。

  更理想的做法是：結合 log1p 的「相對均勻化」效果，
  同時在損失函數中加入對 MAE 的直接約束。

================================================================
五、V4 改進方向（已執行）
================================================================

  V3 的問題：log1p 改善 NMAE 但犧牲 MAE/RMSE/SMAPE
  V4 回到原始空間（不用 log1p），改善學習策略：
  - Layer-wise LR Decay（不同層用不同 LR）
  - 混合損失（0.7×Huber + 0.3×MSE）
  - CosineAnnealingWarmRestarts（定期重啟 LR）
  目標：用更精細的遷移學習技術，改善 MAE 的同時維持 SMAPE

================================================================
六、結論
================================================================

  V3 成功解決了 V2 的 Per-user NMAE 異常問題（410% → 61%），
  但引入了 Log1p 的反轉換放大效應，讓 MAE 和 SMAPE 退步。

  目前最佳指標分布：
  ✅ 最佳 Val MAE     ：V2（4,303 元）
  ✅ 最佳 Test SMAPE  ：V1（78.44%）
  ✅ 最佳 Per-user NMAE：V3（Test 61.4%，Val 73.0%）
  ⚠️  Test MAE        ：V0 仍最佳（5,952 元）

  → 現有技術在不同指標上各有優劣，V4 和 V5 將嘗試整合各版本的優點。
