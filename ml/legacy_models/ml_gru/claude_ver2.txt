一、本版本改動摘要（相對 V1）
----------------------------------------------------------------
V1 的核心問題：hidden=128 導致 16萬參數 vs 4067 筆資料，
finetune 在 epoch 5 就過擬合。

V2 針對過擬合做出以下修正：

  1. 縮小模型容量：hidden_size 128 → 64
     理由：V1 參數/樣本比約 1:25，遠低於推薦的 1:50 以上。
           縮回 64 讓比例回到 1:98，大幅降低過擬合風險。
           模型參數量：160,386 → 41,282（-74%）

  2. Dropout 提高：0.3 → 0.4
     理由：更強的訓練時隨機失活，防止神經元記憶特定樣本。

  3. Weight Decay 提高：1e-4 → 5e-4
     理由：更強的 L2 正則化，懲罰過大的權重值。

  4. 漸進式解凍（Gradual Unfreezing）
     理由：V1 全量微調在 epoch 5 就達到最佳、立即過擬合。
           V2 分兩階段：
           Phase 1（60 epochs）：凍結 GRU 層，只訓練 Attention+FC（2,306 參數）
             → 讓輸出層先適應個人消費的尺度與分布
           Phase 2（140 epochs）：解凍所有層，全量微調（LR=1e-4）
             → 用更小的 LR 微調整個網路，避免 catastrophic forgetting
           效果：V2 Phase 1 在全 60 epochs 都持續改善（未觸發 early stop），
                 Phase 2 持續改善到 epoch 74 才早停，共訓練了充分的輪數。

  保留 V1 的改進：
  - Temporal Attention（對 SMAPE 有幫助）
  - LayerNorm
  - HuberLoss
  - AdamW

三、分析與檢討
================================================================

【正面發現】

  1. Val MAE 和 Val RMSE 達到目前三版中最佳
     Val MAE 4,303（vs V0 4,623、V1 4,583），顯示在驗證期間
     V2 的漸進式解凍確實讓模型在驗證集上更穩定地收斂。
     Val RMSE 也改善了 11%，代表大誤差的情況減少。

  2. 訓練過程更健康
     Phase 1 全 60 epochs 都在持續改善，Phase 2 到 epoch 74 才早停。
     相比 V1 只訓練到 epoch 5 就提早結束，V2 的訓練曲線明顯更正常。

【問題發現】

  1. Test MAE 繼續退步（5,952 → 6,258，+5.1%）
     這是最大的問題。驗證集改善（Val MAE ▼6.9%）但測試集退步，
     典型的 val/test 分布不一致問題。
     推測原因：
     - 驗證集（中間 15%）和測試集（最後 15%）代表不同時期的消費模式
     - 漸進式解凍讓模型更好地適應驗證集的分布，但不一定代表測試集

  2. Test Per-user NMAE 嚴重退步（138% → 410%）
     這是 V2 最大的異常。410% 代表有些用戶的相對誤差極高。
     可能原因：模型對某些特定用戶的預測嚴重偏離，
     而這些用戶恰好在測試集中比例較高。
     此指標對極端值敏感，需要進一步分析是哪些用戶造成的。

  3. Test SMAPE 比 V1 退步（83.9% vs V1 的 78.4%）
     V2 縮小了模型容量，可能減少了 Attention 對消費模式的學習能力，
     導致在相對誤差上不如 V1 的大模型。

================================================================
四、根本原因分析
================================================================

V2 揭示了一個核心矛盾：
  ■ 縮小模型 → 減少過擬合 → Val MAE 改善
  ■ 縮小模型 → 表達能力降低 → Test SMAPE 和 Test MAE 退步

「漸進式解凍」本身是成功的（訓練曲線健康），
但縮小模型容量可能過頭了。
另外，Test Per-user NMAE 410% 的異常值需要調查：
  - 可能是某用戶在測試集中有極端消費（婚禮、旅遊等），
    模型在訓練時從未見過這種極端情況
  - 或者是 log1p 轉換不當（但 V2 沒有用 log1p，所以應不是這個原因）
  - 最可能是資料分布本身的問題：測試集（最後 15% 時間段）
    包含某些用戶的季節性消費高峰

================================================================
五、V3 改進方向（已執行）
================================================================

  基於 V2 的 Test Per-user NMAE 異常，V3 採用：
  - 沿用 V2 架構（hidden=64，漸進式解凍）
  - 加入 Log1p 目標轉換：讓模型在相對尺度上學習，
    理論上改善 SMAPE 和 Per-user NMAE
  - 加入輸入噪聲增強（σ=0.02）：提高訓練多樣性

================================================================
六、結論
================================================================

  V2 在驗證集上達到目前最佳（Val MAE=4,303），
  但測試集指標仍未突破 V0 基線（Test MAE 6,258 vs V0 的 5,952）。

  訓練過程的改善（漸進式解凍、更充分訓練）是正確方向，
  但模型縮小後的表達能力下降抵消了部分收益。

  目前各版本最佳指標：
  ✅ 最佳 Val MAE：V2（4,303 元）
  ✅ 最佳 Test SMAPE：V1（78.44%）
  ⚠️  Test MAE：V0 仍最佳（5,952 元）

  → 代表絕對誤差（MAE）和相對誤差（SMAPE）存在 trade-off，
    V3~V5 將試圖找到兩者都改善的解法。
