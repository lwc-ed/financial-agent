一、本版本改動摘要（相對 V3）
----------------------------------------------------------------
V3 的問題：Log1p 改善了 Per-user NMAE（61%）但犧牲了 MAE 和 SMAPE。
V4 回到原始空間（不用 log1p），改用更精細的遷移學習技術。

V4 的三個核心改動：

  1. Layer-wise Learning Rate Decay（LLRD）
     理由：遷移學習的關鍵問題是：越靠近輸入的層（GRU 底層）
           已學到通用的時序特徵，應該變動越少；
           越靠近輸出的層（FC Head）需要更快適應新任務。
           V4 對不同層設定不同 LR：
             GRU Layer 0  : LR = 5e-6  （幾乎不動）
             GRU Layer 1  : LR = 1e-5  （微調）
             Attention    : LR = 3e-5  （中速）
             FC Head      : LR = 1e-4  （正常速度）
           這比全量統一 LR（V2 的 1e-4）更保護底層特徵，
           避免 catastrophic forgetting。

  2. 混合損失函數（0.7×Huber + 0.3×MSE）
     理由：
     - HuberLoss（delta=1.0）：對大誤差（outlier）較為穩健，
       在高偏態的消費資料中有效抑制少數大額消費的主導效應。
     - MSELoss：直接優化平方誤差，提高模型對一般誤差的敏感度。
     - 混合比例 7:3 讓模型同時兼顧 outlier 穩健性和整體 RMSE 改善。
     V2/V3 純用 HuberLoss，忽略了 MSE 對一般誤差的約束能力。

  3. CosineAnnealingWarmRestarts（T_0=40, T_mult=2）
     理由：學習率定期重啟可以幫助模型跳出局部最優（local optima）。
     T_mult=2 讓每個周期是前一個的 2 倍長：
       第一個周期：40 epochs（快速探索）
       第二個周期：80 epochs（精細搜索）
     在每個 LR 重啟點，模型能以較高 LR 重新探索參數空間，
     理論上找到更好的全局最優。

  其他設定（沿用 V2）：
  - hidden=64, Attention, FC(64→32→1), Dropout=0.4
  - AdamW, weight_decay=5e-4
  - 無 log1p 轉換（原始標準化空間）
  - 無 Gradual Unfreezing（全量微調，但透過 LLRD 保護底層）
  - 使用 V2 pretrain 權重（pretrain_gru_v2.pth）
  - 200 epochs max，patience=30

三、分析與檢討
================================================================

【正面發現】

  1. Test MAE 首次突破 V0 基線（5,773 vs V0 的 5,952，▼3.0%）！
     這是 V1~V4 四個版本中，第一個在 Test MAE 上超越原始 V0 的版本。
     LLRD 的關鍵貢獻：底層 GRU 特徵受到保護（LR=5e-6/1e-5），
     只微調上層，避免了 V1 全量高 LR 造成的特徵破壞。

  2. Test RMSE 也達最佳（14,513 vs V0 的 14,906，▼2.6%）
     混合損失中的 MSE 分量確實幫助降低了大誤差的頻率。

  3. Test SMAPE 在原始空間版本中表現不錯（82.23%）
     比 V0（90.18%）、V2（83.92%）都好，僅次於 V1（78.44%）。

  4. Val RMSE 是去除 V2 外的次佳（10,999）
     LLRD + WarmRestarts 的組合讓訓練更穩定地收斂。

【問題發現】

  1. Test Per-user NMAE 再次飆高（396.20%）
     這是 V4 最大的弱點。不用 log1p 就會回到原始空間的分布問題：
     部分用戶的相對誤差極高，NMAE 指標惡化。
     這重新印證了 V3 的發現：要改善 NMAE 必須用 log1p，
     但 log1p 又犧牲 MAE。

  2. Val Per-user NMAE 也偏高（198.34%，比 V2 的 196% 略差）
     同上，原始空間訓練無法有效控制相對誤差。

  3. Val MAE 退步（4,541 vs V2 的 4,303）
     LLRD 對驗證集性能有一定影響，但不嚴重。
     主要原因：V4 不做 Gradual Unfreezing，直接全量微調（但用 LLRD
     保護），收斂路徑不同於 V2 的分段訓練。

================================================================
四、根本原因分析
================================================================

V4 揭示了一個重要發現：

  MAE/RMSE 最佳化路徑（原始空間）：
  → V4 的 LLRD 成功：Test MAE 5,773（突破 V0）
  → 但 Per-user NMAE 高達 396%（小額用戶相對誤差仍未解決）

  NMAE 最佳化路徑（Log1p 空間）：
  → V3 的 Log1p 成功：Test NMAE 61.4%（極佳）
  → 但 Test MAE 退步至 6,347（比 V0 還差）

  最理想的 V5 方向：
    結合 Log1p（控制相對誤差）+ LLRD（保護底層特徵）
    → 在 log 空間用小 LR 微調底層、大 LR 調整頂層
    → Ensemble 多個模型 + 偏差修正，兼顧 MAE 和 NMAE

================================================================
五、V5 改進方向（已執行）
================================================================

  V4 成功的部分：LLRD 讓 Test MAE 首次超越 V0
  V4 失敗的部分：未能改善 Per-user NMAE（396%）

  V5 嘗試整合所有版本的優點：
  - 3-model Ensemble（seed 42/123/777）：降低隨機性影響
  - Log1p 目標轉換：控制相對誤差（NMAE）
  - LLRD（V4 的成功）：保護底層 GRU 特徵
  - 輸入噪聲增強（V3 的成功）：提高泛化
  - Per-user 偏差修正：利用 val set 估計每個用戶的系統性偏差
  目標：Test MAE < 5,773 且 Test NMAE < 200%

六、結論
================================================================

  V4 是第一個在 Test MAE 和 Test RMSE 上同時超越 V0 的版本！
  LLRD 是關鍵貢獻：透過分層學習率保護遷移學習的底層特徵。
  混合損失（Huber+MSE）也對 RMSE 的改善有貢獻。

  目前最佳指標分布：
  ✅ 最佳 Test MAE  ：V4（5,773 元）← 首次超越 V0
  ✅ 最佳 Test RMSE ：V4（14,513 元）← 首次超越 V0
  ✅ 最佳 Val MAE   ：V2（4,303 元）
  ✅ 最佳 Test SMAPE：V1（78.44%）
  ✅ 最佳 Per-user NMAE：V3（Test 61.4%，Val 73.0%）

  → V4 和 V3 的優勢互補，V5 將嘗試整合兩者。
