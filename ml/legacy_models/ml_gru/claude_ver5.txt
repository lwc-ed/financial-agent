一、本版本改動摘要（整合 V3+V4 優點）
----------------------------------------------------------------
V3 的強項：Log1p 讓 NMAE 極佳（61%），但 MAE 退步
V4 的強項：LLRD 讓 Test MAE 首次超越 V0（5,773），但 NMAE 差（396%）

V5 目標：兼顧 MAE 和 NMAE，整合所有版本的優點。

V5 的四個核心機制：

  1. 3-model Ensemble（seeds 42, 123, 777）
     理由：單一模型的預測容易受隨機性影響。
           不同初始 seed 讓三個模型學到不同的局部特徵，
           取平均可以降低方差（variance），提高預測穩定性。
           Ensemble 是機器學習中最穩健的誤差降低技術之一，
           在小資料集（4,067 筆）中特別有效，
           因為單模型容易在不同 fold 上表現不穩定。

  2. Log1p 目標轉換（沿用 V3）
     理由：V3 已驗證 log1p 對 NMAE 的改善效果（61.4%）。
           V5 在 log 空間訓練，透過 Ensemble 和偏差修正
           嘗試同時維持良好的 MAE。
           三個模型各自在 log 空間訓練，輸出取 expm1 後再平均。

  3. LLRD 分層學習率（沿用 V4）
     理由：V4 已驗證 LLRD 對 Test MAE 的改善效果。
           即使在 log 空間，底層 GRU 仍應用小 LR 保護，
           上層 FC 用大 LR 快速適應 log1p 的新目標分布。
           V5 的 LLRD 設定（同 V4）：
             GRU Layer 0  : LR = 5e-6
             GRU Layer 1  : LR = 1e-5
             Attention    : LR = 3e-5
             FC Head      : LR = 1e-4

  4. Per-user 偏差修正（Val-set Bias Correction）
     理由：Ensemble 在 log 空間取平均後，個別用戶可能有
           系統性偏差（bias）。例如某個用戶的消費習慣特殊，
           模型總是高估或低估。
           偏差修正步驟：
           a) 用 val set 計算每個用戶的 median 預測誤差
              bias_u = median(pred_u - true_u)
           b) 在 test set 的預測上減去這個偏差
              final_pred_u = raw_pred_u - bias_u
           效果：Test MAE 從 6,776 元降至 5,821 元（▼14%！）

  其他設定：
  - 輸入噪聲增強（σ=0.02，沿用 V3）
  - Gradual Unfreezing（Phase 1: 凍結 GRU 60 epochs，
                        Phase 2: 全量微調 140 epochs）
  - AdamW, weight_decay=5e-4, HuberLoss
  - 使用 V2 pretrain 權重（三個 seed 共用同一 pretrain）

三、分析與檢討
================================================================

【突破性正面發現】

  1. Test SMAPE 史上最佳（69.82%）！
     V1 的 78.44% 曾是最佳，V5 提升到 69.82%，▼8.6%！
     Val SMAPE 也達 74.25%（全版本最佳）。
     原因：
     a) Log1p 讓模型在比例空間學習，小額和大額消費預測更均等
     b) 3-model Ensemble 降低了極端預測的機率（平滑效果）
     c) SMAPE 對接近零的預測敏感，但 Ensemble 均值比單模型
        更遠離零邊界，降低了 SMAPE 惡化的情況

  2. Test Per-user NMAE 史上最佳（60.84%）！
     V3 的 61.40% 是前最佳，V5 再降至 60.84%（▼0.56%）。
     Val NMAE 也達 62.74%（比 V3 的 73.02% 好！）。
     原因：
     a) Log1p 的比例均等化效果（同 V3）
     b) Per-user 偏差修正直接修正了每個用戶的系統性誤差
     c) Ensemble 平均降低了單個模型對特定用戶的過擬合

  3. Per-user 偏差修正效果顯著
     在 test 集上，原始 ensemble 預測 MAE = 6,776 元，
     加上偏差修正後降至 5,821 元，整整降低了 954 元（▼14%）！
     這說明每個用戶確實有可修正的系統性預測偏差。

  4. Val MAE 第二佳（4,360 元，僅次於 V2 的 4,303 元）
     整合 Ensemble + Log1p + LLRD 讓驗證集表現大幅優於 V3/V4。

【問題發現】

  1. Test MAE 略遜於 V4（5,821 vs V4 的 5,773）
     V4 在原始空間（無 log1p）優化 MAE 效果更直接。
     V5 雖透過偏差修正讓 MAE 從 6,776 降至 5,821，
     但仍稍遜於 V4 的 5,773。
     原因：Log1p 空間的訓練目標是最小化比例誤差，
           反轉換後的 MAE 不一定全局最優。

  2. Test RMSE 退步（15,299 vs V4 的 14,513）
     同上，log1p 對大額消費的 expm1 放大效應仍然存在。
     Val RMSE 也退步（13,716 vs V4 的 10,999）。

  3. log_target_scaler_v5.pkl 時間戳異常
     scaler 在 01:38 就儲存（V4 完成後立即），
     代表 V5 的三個 seed 共用同一個 log scaler。
     這是正確的設計（同樣的標準化空間），無功能問題。

四、根本原因分析
================================================================

V5 的 Per-user 偏差修正揭示了一個重要洞察：

  個人消費預測存在兩種誤差：
  A. 隨機誤差：每天消費的隨機波動，難以預測
  B. 系統誤差（偏差）：模型對特定用戶的固定高估或低估

  V5 用 val set 估計並修正了 B 類誤差，
  讓 MAE 從 6,776 降至 5,821（-14%），效果非常顯著。
  這說明仍有剩餘的 A 類隨機誤差無法透過現有架構消除。

  SMAPE 大幅改善（69.82%）的主要原因：
  - Log1p 讓模型不再被大額消費主導
  - Ensemble 讓預測值不會極端地趨近於 0 或過高
  - 這兩個效果組合正好對 SMAPE 的計算有利

  MAE vs SMAPE 的 trade-off 仍然存在，但 V5 找到了新的平衡點：
  - V4：Test MAE 5,773（優）, SMAPE 82.23%（中）
  - V5：Test MAE 5,821（次優）, SMAPE 69.82%（最優）

六、結論
================================================================

V5 是目前在 SMAPE 和 Per-user NMAE 上的全面最佳版本。
三模型 Ensemble + Log1p + Per-user 偏差修正的組合，
在相對誤差指標上取得了突破性的改善。

如果以應用場景區分：
  → 需要最小化絕對金額誤差（MAE）：選用 V4（5,773 元）
  → 需要最小化相對誤差（SMAPE/NMAE）：選用 V5（69.82% / 60.84%）
  → 兩者兼顧：V5 仍是最均衡的選擇（Test MAE 5,821 僅略遜 V4）
